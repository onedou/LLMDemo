# LLMDemo 产品规划（v0.2）

> 负责人：产品经理 ｜ 更新日期：2026-07-02 ｜ 状态：v0.2 迭代进行中

## 一、产品定位

**面向谁**：初学 LLM 的学生、培训课程学员、需要给非技术同事演示"LLM 是怎么炼成的"的工程师。

**核心价值**：在一台普通电脑上，用最少的代码完整走通 LLM 四步流程——**数据（data/）→ 训练（trainer.py）→ 推理（inference.py）→ 交互（CLI / Web）**。它是"教学标本"，不是产品级聊天机器人。

**技术底盘（约束即定位）**：
- 对话模型仅 128 维 / 2 层 Transformer，**纯记忆式**：以 `<BOS>问题<SEP>回答<EOS>` 格式精确记住约 100 条问答对，贪心解码复现答案；
- `TRAIN_MAX_SEQ_LEN=64`：中文按字分词后，单条"问题+回答"合计约 60 字以内；
- 无预训练权重，词表级 tokenizer（新词→`<UNK>`），不具备泛化和常识推理能力；
- 实时类问题（时间/日期/算术）靠内置规则技能兜住，不走模型。

## 二、当前迭代 v0.2：中文支持 + 简单问答 + Web UI

**目标**：用户打开浏览器，就能用中文和 Nova 完成一次"像样的"问答演示。

**范围**（进行中）：
1. 中文输入支持：清洗正则保留 CJK 字符，中文按字分词（开发中）；
2. 简单问答：扩充 `data/conversation_data.txt` 中文问答对 + 时间/日期/算术内置规则技能，重训对话模型（开发中）；
3. Web UI：`web_server.py` + `static/index.html`，`POST /api/chat` 接口（开发中）。

**明确不做**：多轮上下文理解、训练外问题的智能回答、并发性能优化。

### 验收标准清单（交付测试工程师）

前置条件：用最新 `data/conversation_data.txt` 重训完成，`python web_server.py` 可启动。

| # | 验收项 | 通过标准 |
|---|--------|----------|
| 1 | 中文身份问答 | 输入"你叫什么名字"，返回含 **Nova** 的中文回答 |
| 2 | 英文身份问答 | 输入 "What's your name?"，返回含 **Nova** 的英文回答 |
| 3 | 中文知识问答（抽查≥5条训练内问题） | 如"中国的首都是哪里"→含"北京"；"一年有多少天"→含"365"；回答与训练数据一致 |
| 4 | 时间技能 | 输入"现在几点"，返回当前系统时间（HH:MM，误差≤1分钟）；英文 "What time is it?" 同样生效 |
| 5 | 日期技能 | 输入"今天几号"，返回的年/月/日/星期与系统日期一致 |
| 6 | 算术技能 | "1加1等于几"→2；"12除以4"→3；"3乘5"→15；"5除以0"→返回除零提示，不崩溃 |
| 7 | 训练外问题兜底 | 输入训练集外问题（如"量子纠缠是什么"），返回非空回复、进程不崩溃、无报错堆栈（内容不要求正确，见风险节） |
| 8 | 输出干净 | 任意回复中不出现 `<BOS>` `<EOS>` `<SEP>` `<PAD>` `<UNK>` 等特殊标记 |
| 9 | 中文标点规范 | 中文回复使用中文标点（，。！？），并以句末标点结尾 |
| 10 | Web 页面加载 | 浏览器访问 `http://127.0.0.1:8000` 返回聊天页面，页面上显示机器人名字 Nova（与 config.py 的 BOT_NAME 一致） |
| 11 | Web 中文收发 | 页面输入中文消息可正常发送并显示中文回复，无乱码；`curl POST /api/chat` 返回 JSON 中中文不被转义为 `\uXXXX` |
| 12 | Web 异常输入 | 发送空消息 / 纯空格：接口返回 400 + 中文错误提示，页面不白屏 |
| 13 | Web 连续对话稳定性 | 连续发送 10 条消息（中英混合），每条均有回复，回复不互相串扰，服务不崩溃 |
| 14 | 数据规范检查 | `data/conversation_data.txt` 每条"问题+回答"编码后总长 ≤ 64 token（超长会被截断导致答非所问，需开发修正数据） |
| 15 | CLI 回归 | `python interface.py interactive` 中文对话正常，与 Web 端回答一致 |

## 三、路线图

### 短期 v0.3（1~2 个迭代内可落地）

| 事项 | 优先级 | 理由 |
|------|--------|------|
| 检索式回答兜底：用户输入与训练问题做相似度匹配（字符 n-gram/编辑距离），相近则返回对应答案，否则明确说"不会" | **P0** | 治本解决"训练外问题胡答"，纯 Python 实现零训练成本，是小模型体验提升性价比最高的一件事 |
| 训练数据自动校验：训练前检查问答对长度超 64 token 即告警 | **P0** | 超长截断是当前最隐蔽的答非所问来源，一个脚本就能堵住 |
| 验收清单脚本化为自动回归测试（扩展 test_conversation.py） | P1 | 每次重训模型后一键回归，测试工程师从手工验证中解放 |
| 流式输出（SSE 逐 token 推送 + 前端打字机效果） | P1 | 演示观感提升明显，且能直观展示"逐 token 生成"的教学点 |
| 对话历史持久化（前端 localStorage + 后端 JSONL 日志） | P1 | 成本低；后端日志还能收集"用户真实问题"反哺语料扩充 |
| jieba 词级分词对照实验（不替换默认按字分词） | P2 | 教学价值（对比分词粒度影响），但小语料下词级易加剧 `<UNK>`，只做实验分支 |

### 中期（3~6 个迭代）

| 事项 | 优先级 | 理由 |
|------|--------|------|
| 两阶段训练：更大中文语料预训练 + 问答微调 | P1 | 让模型有基本语言流畅度，是摆脱"纯记忆"的唯一正路，但需准备语料和更长训练时间 |
| 多轮对话上下文：训练格式改为多轮拼接，`TRAIN_MAX_SEQ_LEN` 扩到 128 | P1 | 依赖上一项（当前模型记忆单轮都吃紧）；先扩序列长度再谈上下文 |
| Docker 化 + 一键启动脚本（训练/服务分离） | P1 | 教学场景常见"环境装不上"，Docker 消除演示第一道门槛 |
| 训练过程可视化面板（loss 曲线实时推送到 Web） | P2 | 强化"训练"环节的教学演示价值 |

### 长期（愿景）

| 事项 | 优先级 | 理由 |
|------|--------|------|
| BPE/子词 tokenizer 替换词表级 tokenizer | P2 | 根治 `<UNK>` 问题，但改动面大（预处理/训练/推理全链路） |
| 模型规模升级档位（如 256d/4层）+ 配置化切换 | P2 | 硬件允许时演示"规模效应"，需配套更大语料才有意义 |
| 课程化：配套 Jupyter Notebook 分章讲解四步流程 | P2 | 把 demo 升级为完整教学材料，扩大受众 |

## 四、风险与限制（对外承诺的边界）

1. **能力边界必须如实告知**：模型只能复现训练过的约 100 条问答；训练外问题的回答不可靠（v0.3 检索兜底前只能"礼貌地不会"）。演示话术、README 均不得宣传为"通用聊天机器人"。
2. **序列长度硬限制**：`TRAIN_MAX_SEQ_LEN=64` 下中文问答合计约 60 字，扩充语料时超长条目会被静默截断，导致训练出"半句话"——这是当前最大的数据质量风险（对应验收项 14）。
3. **无泛化能力**：换个问法（"你是谁"vs"你叫什么名字"）就可能答非所问；缓解手段是同义问法在语料中多写几遍，根治要靠 v0.3 检索兜底。
4. **重训成本**：改机器人名字、增删问答对都需要重新训练；迭代节奏受训练环节制约，需在排期中预留。
5. **服务能力**：Flask 单进程 + 推理全局锁，请求串行处理，仅支持单人演示场景，不承诺任何并发指标。
6. **新词即 `<UNK>`**：词表固定，用户输入生僻字/新词会退化为未知符，影响匹配效果。

## 五、v0.3 预训练里程碑（2026-07-03 完成）

中期路线中的"两阶段训练"第一阶段与长期路线中的"BPE tokenizer"已提前落地：

| 项目 | 结果 |
|------|------|
| 模型 | GPT-2 结构复刻（Pre-LN / 权重共享 / 学习式位置编码），gpt2-demo 预设：8 层 / 8 头 / 512 维，33.87M 参数 |
| 分词器 | Byte-Level BPE，词表 16384，训练语料 200MB 中文维基（`models/bpe_tokenizer/`），根治 `<UNK>` 问题 |
| 预训练语料 | 中文维基百科 pleisto/wikipedia-cn-20230720-filtered，清洗后 504.6MB / 254,545 篇，编码为 127.2M train tokens + 1.285M val tokens |
| 训练 | 3900 步 ≈ 2 epoch，每步 65,536 tokens；本地 M1 中断后迁移至 Google Colab（T4）完成 |
| 结果 | best val loss **3.985**（困惑度 ≈ 53.8，step 3750），checkpoint：`models/gpt2_pretrain/best.pt` |
| 生成质量 | 中文流畅、维基文体连贯；事实性不可靠（属 34M 模型正常水平），不可作为知识问答直接使用 |

**下一步（v0.4 候选）**：在 `best.pt` 上用对话语料微调，替换现有"记忆式"对话模型，获得基本泛化能力。
